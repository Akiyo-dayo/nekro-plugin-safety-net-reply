# 安全网回复

安全网回复用于处理 Nekro Agent 偶发的模型格式漂移问题：模型本应输出 Python 沙盒代码并调用消息发送工具，但上游 API 有时会直接返回自然语言正文。NA 默认会把这段正文当作 Python 代码执行，最终因中文标点、引号或段落文本触发 `SyntaxError`，随后进入模型重试；如果重试仍然返回纯文本，就会反复失败。

本插件会在运行时轻量接管 `limited_run_code`，只在检测到“明显不是 Python、也不像工具调用”的中文自然语言正文时触发。触发后，插件会按 NA 原版常见格式生成代码：

```python
msg = (
    "第一段\n\n"
    "第二段"
)

send_plain_text_response(_ck, msg)
```

这样消息仍然通过 NA 的沙盒工具链发送，并保留执行记录，而不是直接绕过系统发送。

## 工作方式

1. 插件加载时保存 NA 原始的 `limited_run_code`。
2. 插件安装一个包装函数，检查每次待执行的 `code_content`。
3. 若内容没有代码块、无法被 Python 解析、且不像工具调用或 Python 片段，则判定为纯文本回复。
4. 插件移除 `<think>...</think>` 推理块后，将正文改写为 `msg = (...)` 加预制工具调用。
5. 原始沙盒执行流程继续运行。

## 长文本处理

`send_plain_text_response` 会在发送前自动分段，默认单段不超过约 1200 个字符。分段会优先选择段落空行，其次选择普通换行，最后才按长度硬切。所有分段拼接后应与原文完全一致，避免平台或消息服务截断导致回复不完整。

## 适用场景

- 上游兼容 API 偶发返回纯文本。
- 模型重试后仍然没有按 NA 要求输出工具调用代码。
- 沙盒日志中出现自然语言正文被执行，并报 `SyntaxError: invalid character`。

## 边界

- 插件不会接管有效 Python 代码。
- 插件不会接管看起来像工具调用、赋值、导入、函数定义、控制流等 Python 意图的内容。
- 当前检测策略主要面向中文自然语言正文，避免把残缺代码误发送到聊天里。

## 回滚

禁用或删除插件后重启 NA 即可恢复原始行为。插件清理方法也会在卸载时尝试恢复原始 `limited_run_code`。
